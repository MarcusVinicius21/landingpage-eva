# Frontend Design Skill — EVA

## Missão
Esta skill existe para elevar a qualidade visual do frontend e impedir resultados com cara de:
- template genérico
- dashboard SaaS comum
- layout de IA sem direção de arte
- interface fria, dura ou inconsistente
- redesign desnecessário que destrói partes já boas

O foco é produzir interfaces:
- premium
- sofisticadas
- coerentes
- responsivas
- visualmente intencionais
- fiéis à identidade do projeto

## Contexto obrigatório do projeto EVA
Ao trabalhar neste projeto, assuma que o site EVA deve ser:

- um site institucional / landing page
- single-page
- 100% frontend estático
- responsivo
- visualmente premium
- fiel aos materiais de referência do EVA
- sem backend
- sem autenticação
- sem CMS
- sem virar aplicativo funcional

## Prioridades absolutas
Sempre priorize:

1. fidelidade à identidade visual real do EVA
2. preservação do que já está bom no projeto atual
3. composição visual premium
4. consistência entre seções
5. responsividade real
6. legibilidade e hierarquia visual
7. simplicidade elegante em vez de efeitos aleatórios

## Nunca faça isso
Nunca:
- gere estética genérica de IA
- use cara de template SaaS comum
- use blocos com aparência de dashboard corporativo genérico
- adicione efeitos chamativos só para “parecer bonito”
- reescreva tudo do zero sem necessidade
- destrua partes que já estejam boas
- adicione complexidade visual desnecessária
- misture escopo de frontend com backend/app real
- transforme o site em produto funcional do app
- use gambiarras de CSS/Tailwind para “forçar” resultado visual
- mantenha classes suspeitas ou sintaxe quebrada

## Direção visual obrigatória do EVA
A interface deve refletir:

- roxo profundo e quente
- lilás suave
- creme/off-white
- contraste elegante
- cards arredondados
- sombras suaves
- ritmo visual sofisticado
- tom feminino, acolhedor e confiável
- tecnologia sem cara agressiva
- sofisticação sem frieza

Evite:
- excesso de neon
- excesso de blur sem função
- excesso de gradientes chamativos
- excesso de ornamento
- grids pesados
- blocos com “cara de admin panel”

## Tipografia e hierarquia
Sempre buscar:
- títulos fortes e limpos
- corpo legível
- boa largura de texto
- espaçamento generoso
- hierarquia clara entre headline, subtítulo, supporting text e CTA
- equilíbrio entre informação e respiro

## Composição visual
Ao trabalhar landing pages e seções:
- pense em ritmo entre seções claras e escuras
- preserve coerência entre Header, Hero, miolo, CTA final e Footer
- evite vazios artificiais
- evite transições improvisadas
- evite divisores sem função
- prefira composição contínua e intencional

## Mockups e prova visual
Ao desenhar ou codar áreas de showcase:
- prefira mockups elegantes
- prefira provas visuais plausíveis
- mantenha aparência de produto real
- não faça placeholders grosseiros se já houver base suficiente para um mockup melhor
- não invente carrossel ou sistema complexo quando o escopo ainda não pede isso

## Responsividade
Sempre validar:
- mobile pequeno
- mobile médio
- tablet
- desktop
- widescreen

Evitar:
- overflow horizontal
- grids quebrados
- tipografia mal escalada
- CTA esmagado
- header mobile improvisado
- espaçamento excessivo sem função
- seção que fica boa no desktop e ruim no mobile

## Tailwind e implementação
Ao usar Tailwind:
- use sintaxe correta e estável
- não use classes suspeitas
- não force `!important` de forma errada
- preserve consistência entre componentes
- prefira classes claras e legíveis
- evite duplicação desnecessária

Exemplo:
- prefira `!bg-white` em vez de `bg-white!`
- prefira `hover:!bg-white/90` em vez de `hover:bg-white/90!`

## Estratégia de trabalho
Quando receber uma tarefa visual:
1. ler o estado atual do código
2. ler os materiais de referência
3. identificar o que já está bom
4. identificar o que está genérico, quebrado ou inconsistente
5. corrigir só o necessário
6. preservar o que já funciona
7. revisar composição geral
8. validar responsividade
9. só então concluir

## Em caso de conflito
Se houver conflito entre:
- efeito visual chamativo e fidelidade ao EVA -> priorize fidelidade ao EVA
- reescrever rápido e preservar o que está bom -> priorize preservar
- inventar solução nova e respeitar o plano do projeto -> priorize o plano
- mais elementos visuais e clareza -> priorize clareza
- complexidade e elegância -> priorize elegância

## Resultado esperado
O frontend final deve parecer:
- profissional
- premium
- coeso
- intencional
- alinhado ao universo visual do EVA
- superior a um template adaptado
- pronto para revisão humana com o mínimo de retrabalho
